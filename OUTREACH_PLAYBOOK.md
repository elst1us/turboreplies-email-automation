# TurboReplies Outreach Playbook

The repeatable system for cold outreach. Today the **first email is sent by hand**
(better deliverability from a real mailbox, room to personalize); the
**follow-ups and reply detection are automated** by this repo. The end goal is to
automate the whole loop — see [Automation roadmap](#automation-roadmap).

The email copy is generated by `src/templates.ts` and is the single source of
truth. Manual drafts come from `npm run draft`, so the hand-sent first touch and
the automated follow-ups never drift apart.

## 1. Who to target (ICP)

Best fit = local businesses with high inbound message volume on
web/Instagram/WhatsApp, high value per lead, year-round demand, able to pay
€49–179/mo. Ranked:

1. **Aesthetic & dental clinics** (`Vertical = clinic`) — Instagram-driven,
   high-ticket patients, multilingual medical-tourism inbound.
2. **Real estate agencies** (`Vertical = real-estate`) — "first to reply wins",
   foreign-buyer agencies especially (multilingual edge).
3. **Short-term-rental property managers** (`Vertical = hotel`) — multi-unit, not
   single rooms; reuses the hospitality angle with real budget.

Avoid: seasonal single-room lodging, walk-in-only low-ticket shops.

### Clinic note: complement, don't replace
Many Italian clinics already run a practice-management suite (e.g. **AlfaDocs**)
that handles structured online booking + reminders. Do **not** pitch booking — it
reads as redundant. Pitch the gap booking software leaves: **conversational,
multilingual pre-booking messages on WhatsApp/Instagram**, qualified and routed
to their existing booking system. The clinic copy is written this way.

## 2. Prep the sheet (per lead)

Fill these columns before drafting:

- `Property`, `Village`/`Location` — business name + place.
- `Owner` — first name(s); enables "Buongiorno Marco" / "Vi scrivo" for pairs.
- `Email` — the real address (verify on their site; do not guess).
- `Language` — `it`/`en`/`fr`/`de`/`es`; sets body language (IT or EN) and the
  demo link locale.
- `Vertical` — `clinic` / `real-estate` / `hotel` (blank ⇒ hotel).
- `Hook` — **one specific, personal line.** This is the highest-leverage field.
  Reference their Instagram, a service, their booking tool, foreign-guest reviews,
  etc. Example: *"Ho visto che gestite le prenotazioni con AlfaDocs: l'assistente
  si affianca e risponde ai messaggi su Instagram, anche in inglese."*

## 3. Send the first email (manual, now)

```bash
npm run draft
```

Prints subject + body for every contactable lead (has `Email`, not `Replied?`,
not `Do Not Contact?`). Copy-paste into your mailbox and send. Sends nothing,
writes nothing.

After sending a row by hand, set in the sheet so the automation takes over the
follow-ups: `Status = Active`, `Date sent = today`, `Follow-up Step = 1`,
`Next Follow-up = today + 3 days`.

What the first email does (by design): leads with the real value, embeds the
locale + vertical demo deep link with UTM tags, and closes with a low-friction
CTA ("send me a question you often get and I'll show you how the assistant
replies") instead of asking permission.

## 4. Follow-ups + replies (automated, now)

`npm run send` (typically run daily) handles the rest:

- Follow-up cadence: step 1 at +3 days, step 2 at +7, step 3 (break-up) at +14.
- Each follow-up re-states one benefit and re-links the demo.
- With `CHECK_REPLIES=true` it checks IMAP first; a detected reply flips the row
  to `Needs reply`, stops follow-ups, and you take over the conversation.

Use `npm run dry-run` to preview what `send` would do without sending.

## Automation roadmap

What's automated vs. manual today, and how to close the gap:

| Stage | Today | To fully automate |
| --- | --- | --- |
| Lead sourcing | Manual research | Google Maps / IG scrape → rows (name, site, IG, phone) |
| Email + vertical + language | Manual cell entry | Infer vertical from category; detect language from site |
| `Hook` line | Hand-written | LLM drafts a per-lead hook from the site/IG, human approves |
| First email | **Manual** (`npm run draft`) | Flip to `npm run send` once warm-up + deliverability are proven |
| Follow-ups | **Automated** | — |
| Reply detection | **Automated** (IMAP) | Add reply triage / suggested response |
| Scheduling | Manual run | Cron `npm run send` daily |

Recommended next build: a lead-intake script (scrape → enrich → LLM-drafted
`Hook` → write rows) so the only human step becomes approving hooks. Until cold
deliverability from the sending domain is proven, keep the **first** email manual.

## Before scaling

- Verify **SPF / DKIM / DMARC** on the sending domain; warm it up gradually.
- Run **50+ leads per vertical** before judging reply rate (n=8 tells you nothing).
- Peak season hurts hospitality — run that vertical Nov–Feb only.
